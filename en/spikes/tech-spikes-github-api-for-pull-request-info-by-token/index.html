Github API for pull request info by token

Background

Section titled Background

Currently, for some users who do not use buildKite deployment, we will migrate some data originally obtained from buildKite to GitHub, so we need to understand the relevant API of GitHub.

Expect

Section titled Expect

We want to use the GitHub API to retrieve the key information we need. With the data obtained this way, we can calculate pr lead time.

2. Get all repos for organization

Section titled 2. Get all repos for organization
2.1 API
Section titled 2.1 API
curl -L \
-H "Accept: application/vnd.github+json" \
-H "Authorization: Bearer <YOUR-TOKEN>" \
-H "X-GitHub-Api-Version: 2022-11-28" \
https://api.github.com/orgs/<ORG>/repos
2.2 Path parameters
Section titled 2.2 Path parameters
  • org:
    • Required
    • string
    • The organization name. The name is not case sensitive.
2.3 Request header
Section titled 2.3 Request header
  • accept
    • Setting to application/vnd.github+json is recommended.
2.4 Query parameters
Section titled 2.4 Query parameters
  • type
    • string
    • Specifies the types of repositories you want returned.
    • Can be one of: all, public, private, forks, sources, member
    • Default: all
  • sort
    • string
    • The property to sort the results by.
    • Can be one of: created, updated, pushed, full_name
    • Default: created
  • direction
    • string
    • The order to sort by
    • Can be one of: asc, desc
    • Default: asc when using full_name, otherwise desc.
  • per_page:
    • integer
    • The number of results per page (max 100)
    • Default: 30
  • page:
    • integer
    • The page number of the results to fetch.
    • Default: 1

4. Get all pull requests by base branch

Section titled 4. Get all pull requests by base branch
4.1 API
Section titled 4.1 API
curl -L \
-H "Accept: application/vnd.github+json" \
-H "Authorization: Bearer <YOUR-TOKEN>" \
-H "X-GitHub-Api-Version: 2022-11-28" \
https://api.github.com/repos/<OWNER>/<REPO>/pulls
4.2 Path parameters
Section titled 4.2 Path parameters
  • owner:
    • Required
    • string
    • The account owner of the repository. The name is not case sensitive.
  • repo:
    • Required
    • string
    • The name of the repository without the .git extension. The name is not case sensitive.
4.3 Request header
Section titled 4.3 Request header
  • accept
    • Setting to application/vnd.github+json is recommended.
4.4 Query parameters
Section titled 4.4 Query parameters
  • state
    • string
    • Either open, closed, or all to filter by state.
    • Default: open
    • Can be one of: open, closed, all
  • head
    • string
    • Filter pulls by head user or head organization and branch name in the format of user:ref-name or organization:ref-name.
    • For example: github:new-script-format or octocat:test-branch.
  • base
    • string
    • Filter pulls by base branch name.
    • Example: main
  • per_page:
    • integer
    • The number of results per page (max 100)
    • Default: 30
  • page:
    • integer
    • The page number of the results to fetch.
    • Default: 1
  • sort:
    • string
    • What to sort results by. popularity will sort by the number of comments. long-running will sort by date created and will limit the results to pull requests that have been open for more than a month and have had activity within the past month.
    • Default: created
    • Can be one of: created, updated, popularity, long-running
  • direction:
    • string
    • The direction of the sort.
    • Default: desc when sort is created or sort is not specified, otherwise asc.
    • Can be one of: asc, desc
